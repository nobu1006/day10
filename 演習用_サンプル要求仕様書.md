# 社内図書貸出システム システム要求仕様書

## 1. 概要

### 1.1 目的

本ドキュメントは、社内の図書・資料の貸出管理を行う「社内図書貸出システム」が満たすべき要件を定義する。

### 1.2 システムの概要

- **システム名**: 社内図書貸出システム
- **対象組織**: 社内の部署・チーム（想定利用者: 最大200名）
- **主な目的**: 図書の貸出・返却の記録、在庫の把握、貸出期限の管理

### 1.3 用語定義

| 用語 | 定義 |
|------|------|
| 図書 | 本システムで管理する対象。紙の書籍および社内規定に基づき登録された資料を含む |
| 利用者 | システムを利用する社員。社内IDで一意に識別される |
| 貸出 | 図書を利用者に一定期間引き渡すこと |
| 返却 | 貸出した図書をシステム上で返却済みとすること |

---

## 2. 機能要件

### 2.1 利用者管理

- **利用者登録**: 社員番号・氏名・所属部署を登録する。登録は管理者のみが行える。
- **利用者検索**: 社員番号または氏名で利用者を検索できる。
- **利用者一覧**: 登録済み利用者の一覧を表示できる。一覧は所属部署で絞り込み可能とする。

### 2.2 図書管理

- **図書登録**: 図書ID、タイトル、著者、ISBN（任意）、カテゴリ、貸出可能数を登録する。登録は管理者のみが行える。
- **図書検索**: タイトル・著者・カテゴリ・キーワードで検索できる。
- **図書一覧**: 登録済み図書の一覧を表示できる。在庫あり/貸出中でフィルタ可能とする。
- **図書削除**: 貸出中のものが1冊もない場合に限り、図書を削除（無効化）できる。削除は論理削除とし、履歴は残す。

### 2.3 貸出・返却

- **貸出処理**: 利用者と図書を指定して貸出を行う。1利用者あたり同時貸出は最大5冊までとする。
- **返却処理**: 貸出IDまたは図書＋利用者を指定して返却を行う。返却日はシステム日付を記録する。
- **貸出期限**: 貸出日から14日間を標準の貸出期限とする。期限超過時は一覧で強調表示する。
- **延長**: 同一図書について、返却期限日前に限り1回のみ14日延長を許可する。

### 2.4 管理者機能

- **管理者ロール**: 利用者管理・図書管理・貸出履歴の参照を実行できるロールを「管理者」とする。
- **一般利用者ロール**: 自分の貸出・返却および図書検索・一覧のみ実行できるロールを「一般」とする。
- **ログ**: 貸出・返却・図書登録・削除の操作はログに記録する（誰が・いつ・何をしたか）。

---

## 3. 非機能要件

### 3.1 パフォーマンス

- **検索応答**: 図書検索・利用者検索の応答時間は、通常負荷時3秒以内とする。
- **同時利用**: 想定同時利用者数20名のとき、上記応答時間を満たすこと。

### 3.2 セキュリティ

- **認証**: 利用は社内認証（ID/パスワード）によるログイン後のみ可能とする。
- **認可**: 管理者のみが利用者管理・図書管理・全貸出履歴の参照を実行できること。一般利用者は自分の貸出情報のみ参照可能とする。
- **個人情報**: 利用者情報（氏名・所属）は社内利用に限定し、外部に送信しないこと。

### 3.3 可用性・運用

- **稼働時間**: 平日 8:00〜20:00 の稼働を想定する。メンテナンスは事前告知の上、月1回以内とする。
- **バックアップ**: 貸出・返却・図書マスタのデータは日次でバックアップを取得する。

### 3.4 ユーザビリティ

- **画面**: 主要な操作（貸出・返却・検索）は3クリック以内で実行できること。
- **メッセージ**: エラー時は「何が問題か」「どう修正すればよいか」が分かるメッセージを表示すること。

---

## 4. 制約・前提

- **利用環境**: 社内ネットワーク上のPCブラウザ（Chrome、Edge 最新版）から利用することを前提とする。
- **データ**: 初年度は図書データ約500件、利用者約100名を想定する。
- **法・規定**: 社内情報管理規定および図書管理規程に従うこと。
