# 社内研修・セミナー申込管理システム システム要求仕様書

## 1. 概要

### 1.1 目的

本ドキュメントは、社内の研修・セミナーの企画・申込・実施管理を行う「社内研修・セミナー申込管理システム」が満たすべき要件を定義する。

### 1.2 システムの概要

- **システム名**: 社内研修・セミナー申込管理システム
- **対象組織**: 全社（想定利用者: 最大2,000名、同時申込ピーク時 100名程度）
- **主な目的**: 研修・セミナーの登録・公開、受講申込・キャンセル、定員・待ち合わせ管理、講師・会場の割当、受講履歴・レポート出力

### 1.3 用語定義

| 用語 | 定義 |
|------|------|
| 研修 | 本システムで管理する対象。 eラーニング・集合研修・セミナーを含む。開催日時・会場・定員・講師が紐づく |
| 受講者 | 研修に申し込む社員。社内IDで一意に識別される |
| 講師 | 研修を実施する社員または外部講師。研修に1名以上割り当てられる |
| 研修管理者 | 研修の企画・登録・申込承認を行うロール。部門ごとに設定可能 |
| 申込 | 受講者が研修の受講を申し込むこと。定員内は「確定」、超過時は「待ち合わせ」となる |
| 会場 | 集合研修を実施する場所。システムに登録され、研修と紐づく |

## 2. 機能要件

### 2.1 ユーザー・ロール管理

- **ユーザー登録**: 社員番号・氏名・メール・所属部門を登録する。人事システム連携または手動登録。同一社員番号の重複登録は不可。
- **ロール**: 受講者（一般）、講師、研修管理者、システム管理者の4ロール。1ユーザーが複数ロールを持つ場合あり（例: 受講者かつ講師）。
- **講師登録**: 講師として登録されたユーザーは、研修に「講師」として割り当て可能。講師登録は研修管理者またはシステム管理者が行う。
- **研修管理者の範囲**: 部門単位で「その部門向け研修」の管理者を設定できる。全社向け研修はシステム管理者または指定された研修管理者が管理。

### 2.2 会場・施設管理

- **会場登録**: 会場コード、名称、所在地、収容人数、設備（プロジェクター、ホワイトボード等）を登録する。
- **会場予約との整合**: 同一会場・同一日時の重複登録は警告し、上書き時は確認を求める。外部の会場予約システムとは連携しない（本システム内のみ）。
- **会場一覧・検索**: 収容人数・設備・所在地で絞り込み検索できる。

### 2.3 研修の企画・登録

- **研修登録**: 研修コード、タイトル、種別（eラーニング / 集合研修 / セミナー）、概要、対象者（全社 / 部門指定）、定員、申込開始日・終了日、開催日時（複数日対応）、会場（集合の場合）、講師（1名以上）を登録する。
- **研修の下書き**: 登録途中の研修は「下書き」で保存でき、公開前に内容を編集できる。公開後は申込状況に応じて編集を制限（申込者がいる場合は項目により変更不可とする）。
- **研修の公開・非公開**: 公開すると受講者に一覧表示され申込可能。非公開にすると一覧から隠し、既存申込はキャンセル扱いとするか運用で規定する。
- **複数回開催**: 同一タイトルの研修を、異なる日時・会場で複数回登録できる（例: 新入社員研修 4月・10月）。

### 2.4 申込・キャンセル・待ち合わせ

- **申込**: 受講者は公開中の研修に対して申し込む。定員内であれば申込は「確定」、定員超過の場合は「待ち合わせ」となる。待ち合わせは確定者がキャンセルした場合、申込日時順に繰り上げ可能とする。
- **申込制限**: 同一研修への重複申込は不可。同一日時帯に重複する他研修に確定申込している場合、申込時に警告を表示する（ブロックするかは運用で規定）。
- **キャンセル**: 確定申込は、開催日のN日前まで（例: 3営業日）キャンセル可能。キャンセル時、待ち合わせがいる場合は自動で繰り上げ処理を行う。
- **繰り上げ通知**: 待ち合わせから確定に繰り上げられた場合、受講者にメール通知する。

### 2.5 受講履歴・成績・修了証

- **受講履歴**: 受講者ごとに、申込・確定・キャンセル・欠席・修了の履歴を一覧表示する。研修管理者は自部門の受講状況を集計できる。
- **修了登録**: 集合研修・eラーニングの修了は、講師または研修管理者が登録する。修了日・成績（合否または点数）を記録できる。
- **修了証**: 修了した研修について、PDF形式の修了証を発行できる。発行履歴を残す。

### 2.6 通知・リマインダー

- **申込受付通知**: 申込確定時、受講者にメール通知する。
- **リマインダー**: 開催の前日またはN日前に、確定申込者へリマインダーメールを送信する。
- **講師・研修管理者向け**: 新規申込・キャンセル・繰り上げが発生した場合、該当研修の研修管理者にメール通知する（設定でON/OFF可能）。

### 2.7 集計・レポート

- **部門別受講集計**: 部門・期間を指定し、研修種別ごとの申込数・修了数を集計する。
- **研修別集計**: 研修ごとに、申込数・確定数・待ち合わせ数・キャンセル数・修了数を集計する。
- **CSVエクスポート**: 受講履歴・申込一覧をCSVでエクスポートできる。システム管理者および研修管理者（自部門分）が実行可能。

### 2.8 システム管理者機能

- **全マスタの登録・編集・削除**: ユーザー、ロール、会場、研修（全社分）の参照・登録・編集・削除。
- **ログ参照**: 申込・キャンセル・繰り上げ・修了登録・ログイン等の操作ログを参照できる。保持期間は1年を想定。

## 3. 非機能要件

### 3.1 パフォーマンス

- **一覧表示**: 研修一覧・申込一覧の初期表示は3秒以内とする。検索・フィルタ適用後の表示も同様。
- **同時申込**: 定員が残り少ない研修への同時申込が 50件/秒 程度発生した場合でも、定員超過・二重確定が発生しないこと（トランザクションと在庫制御で保証）。
- **集計・レポート**: 1年分の履歴を対象とした集計は 30秒以内を目安とする。

### 3.2 セキュリティ

- **認証**: 社内ID/パスワードまたはSSOによる認証。パスワードはポリシーに従い（長さ・複雑さ・有効期限）管理する。
- **認可**: ロールに応じた画面・機能の制御。受講者は自社の研修一覧・自社の申込履歴のみ。研修管理者は担当範囲の研修・申込のみ。システム管理者は全データにアクセス可能。
- **個人情報**: 氏名・メール・所属は社内利用に限定し、外部に送信しない。CSVエクスポートはダウンロード権限を持つ者のみ実行可能とし、ログに記録する。

### 3.3 可用性・運用

- **稼働時間**: 平日 7:00〜22:00 の稼働を想定。メンテナンスは事前告知の上、月1回以内・2時間以内とする。
- **バックアップ**: マスタ・申込・履歴データは日次でバックアップを取得する。保持期間は 90日間。
- **監視**: 申込処理の失敗・通知送信失敗はアラートとし、運用担当が確認できるようにする。

### 3.4 ユーザビリティ

- **画面**: 主要な操作（研修検索・申込・キャンセル）は3クリック以内で実行できること。
- **メッセージ**: 定員超過・重複申込・キャンセル期限超過など、エラー時は理由と取るべきアクションを明示する。
- **多言語**: 初期リリースは日本語のみ。将来の英語対応は項目ラベル・メッセージの外部リソース化で対応可能な設計とする。

---

## 4. 制約・前提

- **利用環境**: 社内ネットワーク上のPCブラウザ（Chrome、Edge 最新版）および社内モバイル端末からの利用を想定する。
- **データ規模**: 初年度は研修 約200件/年、申込 約5,000件/年、ユーザー 約2,000名を想定する。
- **外部連携**: 初版では人事システム・会場予約システムとの連携は行わない。必要に応じて将来、API連携を検討する。
- **法・規定**: 社内個人情報管理規定および研修管理規程に従うこと。
